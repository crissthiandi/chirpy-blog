How to create a Pull Request (PR) on GitHub

Siddharth Arumugam on Jul 82021-07-08T13:30:00+05:30
1 min read

In this post, we will see how to create a Pull Request on GitHub.

For this example, I will be making a PR to the discord assistant of Hack Armour.

Fork the repository

First step, fork the repository.

Clone the forked repository

Clone the forked repo locally:

1

$ git clone https://github.com/{user}/{repoName}.git

Switch branch

Usually you don’t want to mess with the main/master branch. Switch branches before adding your changes.

1

$ git checkout -b {branchName}

Some pointers for naming the branch:

  1. Use your username: Using your username ensures that your changes don’t conflict with other branches. (e.g.) mrnightdev-patch-5

  2. Include the name of the feature: For example let’s say I am adding a moderation feature, add-mod-cmds could be a way to name my branch.

Commit the changes

Make the necessary changes and commit them.

1
2
3

$ git add . # This tells git to stage your changes.
$ git commit -m "Your message here"
$ git push # Push your changes to the forked repo

If you created a new branch locally, you will need to set the local branch to fetch its upstream from GitHub.

1
2
3

$ git remote
origin
$ git push -u origin {branchName} # replace origin with the name of your remote

Creating the pull request

Open your forked repo on GitHub.

Switch branches

In the contribute drop down box, click Open pull request.

Give your PR a good name and description.

Submit the pull request.

🎉 You have successfully created a Pull Request!
